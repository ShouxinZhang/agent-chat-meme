# Task Log Recorder ZH

用于把一次实际任务沉淀成仓库内的中文任务日志。

## 什么时候用

- 用户明确要求“记录 log”“补任务日志”“把本次任务写入 `.agents/logs/...`”
- 用户给出了类似 `.agents/logs/time(yyyymmdd)/time(精确到秒)+task_name` 的路径约定
- 用户希望日志里固定包含任务摘要、执行动作和改动文件

## 路径规则

1. 日期目录：

```text
.agents/logs/timeYYYYMMDD/
```

2. 日志文件：

```text
timeHHMMSS+task_name.md
```

3. `task_name` 用简短英文、拼音或短横线命名，避免空格。

## 推荐步骤

1. 用 `date '+%Y%m%d %H%M%S'` 取当天日期和精确到秒的时间。
2. `mkdir -p .agents/logs/timeYYYYMMDD`
3. 按 `timeHHMMSS+task_name.md` 创建日志文件。
4. 用中文写日志；如果日志是补记，内容以“实际已经发生的事”为准，不写未来计划。

## 必含内容

日志至少包含下面 4 项：

1. 用户原始 query
2. 你的精简理解
3. 你实际做了什么
4. 改动了哪些文件

## 写法要求

- “用户原始 query”保留原意，必要时可用引用块逐行记录。
- “你的精简理解”只写执行口径，不重复长背景。
- “你实际做了什么”按完成顺序写，优先写真实动作和验证结果。
- “改动了哪些文件”要明确区分：
  - 改了源码文件
  - 只生成了环境、产物或日志文件
- 如果源码文件无改动，要明确写“源码文件无改动”。

## 推荐模板

```md
# Task Log

## 用户原始 query

> ...

## 你的精简理解

...

## 你实际做了什么

1. ...
2. ...
3. ...

## 改动了哪些文件

源码文件无改动。

本次新增/生成的主要路径：

- `...`
- `...`
```

## 注意

- 默认写 Markdown。
- 如果仓库里已经有同类日志，优先延续已有标题和段落命名。
- 日志是对任务的落盘记录，不是新的任务计划。
