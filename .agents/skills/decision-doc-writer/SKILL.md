# Decision Doc Writer

把普通说明文改写成“问题拆解 + 推荐答案”文档。

默认调用.agents/skills/sandbox-workmode技能.

## 什么时候用

- 用户要的是可执行结论，不是背景科普；
- 用户在问“应该怎么选”“到底需要几个”“要不要拆”；
- 用户明确认可“原始问题 -> 多个选择题 -> 给出答案”的写法。

## 规则

1. 先写原始问题。
2. 再拆成若干个决策题。
3. 每题直接给答案，不铺垫长背景。
4. 只写会影响选择的信息。
5. 内部推理默认不写。
6. 最后给最短的最终建议。

## 推荐结构

```md
# <Title>

原始问题：

> ...

## Q1. ...？

答案：...

## Q2. ...？

答案：...

## Q3. ...？

答案：...

## 最终建议

1. ...
2. ...
3. ...
```

## 交付前自检

- 是否每节都在回答一个明确问题？
- 是否前两屏内就能看到默认方案？
- 是否删掉了不影响决策的解释？
